================================================================================
OVERALL CONCLUSIONS
================================================================================

1. WHAT WORKS:
   - LoRA (r=16) + GT AE track supervision -> consistent ~34% 2D ATE reduction
   - Generalisation is genuine (val matches train improvement)
   - z.detach() is a necessary fix for any reproj-type loss

2. WHAT DOES NOT WORK:
   - COLMAP camera supervision on low-parallax -> 3D reproj degrades
   - Reproj loss with noisy COLMAP init -> always saturated, zero gradient signal
   - ViPE depth supervision -> confidence collapse, gradient spikes
   - Both EXP-01 and EXP-02 finetuned models are WORSE than vanilla in 3D reproj

3. QUANTITATIVE SUMMARY:
                  2D track ATE (val)    3D reproj_ATE (val)
   vanilla              2.05px               1.74px
   EXP-01 finetuned     1.34px  (-34%)       5.78px  (+232%)  <- 2D better, 3D worse
   EXP-02 finetuned     1.50px  (-27%)      238.89px (+13627%) <- catastrophic 3D

4. ROOT CAUSE (both experiments):
   Low-parallax -> poorly conditioned COLMAP reconstruction ->
   noisy camera labels -> supervising camera head with these labels
   actively hurts 3D geometry. Track head optimises correctly but
   camera head is pulled toward wrong camera poses.

================================================================================
NEXT: EXP-03 — Self-supervised plane rigidity
================================================================================

Goal: improve cameras WITHOUT relying on COLMAP or ViPE labels.

Core idea — Cross-frame plane rigidity:
  The AE logo tracks lie on a rigid plane. For any two frames:
    1. Unproject GT tracks using predicted depth + predicted camera_i
       -> fit plane pi_i in world space
    2. Unproject GT tracks using predicted depth + predicted camera_j
       -> fit plane pi_j in world space
    3. Loss = disagreement between pi_i and pi_j
  This jointly constrains depth + cameras to be self-consistent
  with no external supervision required.

Why this works for low-parallax:
  - Plane fit from 256 tracks is highly overdetermined -> stable
  - Works regardless of parallax magnitude
  - Cannot be satisfied by degenerate cameras (wrong R or T breaks agreement)
  - No COLMAP, no ViPE needed

Why not freeze depth head:
  - Aggregator features change via LoRA -> depth predictions drift anyway
  - Pretrained VGGT depth init is strong -> implicit regularisation vs collapse
  - Plane rigidity + pretrained init together are sufficient
  - Simpler: no external pseudo-GT at all beyond GT tracks

Proposed supervision (EXP-03):
  camera loss     : weight=0.0   (drop COLMAP)
  depth loss      : weight=0.0   (drop ViPE)
  reproj loss     : null         (drop)
  track loss      : weight=1.0   (keep, high quality signal)
  plane_rigidity  : weight=TBD   (new, to be implemented)

First run (OLD pod — dawy-vggt-exp05-tqmft-8hlbh):
  Launched with OLD code (confidence exploit NOT fixed).
  train/loss_conf_depth   : +0.45 (E0) -> -0.135 (E1) -> -0.680 (E2) -> -1.231 (E2+)
  Grad/depth              : 3.4 (E1) -> 22.3 (E2) — exploding
  Grad/track              : 20.7 (E2)
  KILLED at ep~3: identical failure mode to EXP-03a/03b.

Root cause confirmed:
  Kendall uncertainty loss exploit: loss_conf_depth → -1.23 by ep2.
  Pattern now observed 4/4 times across all experiments with depth enabled.
  The exploit appears regardless of depth weight (0.1 in EXP-03, 0.5 here).
  Structural fix needed in loss.py.

Code fix (training/loss.py):
  Detach loss_reg in confidence term:
    BEFORE: loss_conf = gamma * loss_reg * conf[mask] - alpha * torch.log(conf[mask])
    AFTER:  loss_conf = gamma * loss_reg.detach() * conf[mask] - alpha * torch.log(conf[mask])
  Effect: breaks circular exploit. Confidence head tracks prediction quality
          (via detached loss_reg) without feeding back into prediction head.
          Prediction head still trains via loss_reg_depth (returned separately).
  Committed: d4027b7 "Fix confidence exploit: detach loss_reg in confidence term"

Second run (NEW pod — dawy-vggt-exp05-tqmft-fqmtc):
  Launched with FIXED code.
  train/loss_conf_depth   : +0.093 (step 7) -> +0.001 (step 50) — fix held early
  train/loss_conf_depth   : -1.651 (step 324, ep6) — EXPLOIT RETURNED
  train/loss_conf_depth   : -1.803 (step 348, ep6) — accelerating
  train/objective         : -0.088 (step 419, ep8) — objective went negative!
  val/loss_track          : 0.388 (ep8) — catastrophic, model destabilised
  Grad/track              : 32.73 (ep8, HIGH)

Root cause — detach fix insufficient:
  loss_reg.detach() breaks the circular gradient path but does NOT prevent the
  exploit solution from being reached. The model still drives loss_reg → 0 via
  the separate loss_reg_depth objective term. Once loss_reg is small (even
  detached), the optimal confidence conf* = alpha/(gamma*loss_reg_detached)
  becomes large, and loss_conf = gamma*loss_reg_detached*conf - alpha*log(conf)
  goes deeply negative. The gradient of the confidence head still descends toward
  this negative minimum — it just no longer drags the prediction head with it.
  The exploit held for ~50 steps then reasserted as depth predictions improved.

Action: KILLED at ep8. Detach-only fix is insufficient for long runs. The Kendall
        formulation has a structurally unbounded negative minimum whenever
        loss_reg → 0. A harder fix is needed (clamp loss_conf ≥ 0, or drop
        Kendall entirely). Deprioritised in favour of EXP-06 (depth=null avoids
        the issue entirely while reproj provides the 3D supervision).

        SummaryWriter holds a file handle that becomes stale (ESTALE).
   Fix: Added errno.ESTALE catch in TensorBoardLogger.log(), with
        _recreate_writer() fallback using a new UUID filename suffix.

5. Confidence exploit in Kendall uncertainty loss (loss.py)
   Bug: loss_conf = gamma*loss_reg*conf - alpha*log(conf) creates a circular
        exploit. Better predictions → higher optimal conf → more negative loss
        → incentive for better predictions. Observed as loss_conf_depth → -2.8
        across 4/4 experiments with depth enabled (EXP-01, EXP-03a, EXP-03b,
        EXP-05 old pod).
   Fix: loss_conf = gamma*loss_reg.detach()*conf - alpha*log(conf).
        Confidence head tracks prediction quality without feeding back.
        Prediction head still trains via loss_reg_depth (separate objective term).
   Commit: d4027b7

EXP-06  rembrand_lora_r16_reproj_plane_v1  [LAUNCHED]
================================================================================

Goal: True 3D self-supervision via reprojection loss + plane rigidity.
      First experiment where Grad/depth > 0 is expected from 3D geometry.

Core insight from EXP-04/04b diagnosis:
  The plane_rigidity consistency_weight term has a degenerate minimum: the model
  can collapse depth to a constant across frames and get loss ≈ 0 without ever
  learning correct absolute depth. This is why Grad/depth = 0.000 in both
  EXP-04 and EXP-04b despite the loss mathematically flowing through depth.

  The reprojection loss does NOT have this degeneracy:
    unproject(gt_track_0, pred_depth_0, K_0, E_0) → P_world
    reproject(P_world, K_j, E_j) must match gt_track_j
  Collapsing depth to a constant makes reprojection fail unless cameras also
  collapse — but that breaks the track loss. The constraint is coupled and
  cannot be gamed by any single-head collapse.

Why reproj was unstable before (EXP-02):
  Reproj + COLMAP camera loss = two conflicting camera supervision signals.
  COLMAP labels are noisy for low-parallax → pulls cameras wrong direction
  while reproj tries to correct → large opposing gradients → instability.
  This experiment drops camera loss entirely (camera: null).

Config  : camera=null, depth=null, track.weight=1.0, reproj.weight=0.1,
          plane_rigidity.weight=0.3, consistency_weight=1.0, plane_weight=0.0
          track.max_norm=0.1, depth.max_norm=0.3, camera.max_norm=0.1
          aggregator.max_norm=0.5
Dataset : tracking_whisper_clean_dataset
Commits : eb9ae5a, a2c3ede

Weight rationale:
  reproj.weight=0.1    — conservative; reproj loss is in raw pixels (5-20px
                         early training) so 0.1×10px = 1.0 contribution.
  track.weight=1.0     — raised from earlier 0.5 to balance reproj pixel scale;
                         keeps 2D anchor competitive at ~1.0 contribution.
  plane_rigidity=0.3   — secondary regularizer; once reproj drives real 3D
                         geometry, consistency adds cross-frame constraint.
  camera=null          — no COLMAP supervision (root cause of EXP-02 failure)
  depth=null           — Kendall confidence exploit not yet structurally solved

Gradient clip rationale:
  camera.max_norm=0.1  — very tight; no prior data on reproj camera grad scale
  depth.max_norm=0.3   — tighter than EXP-04b (0.5); reproj drives depth harder
  track.max_norm=0.1   — same as EXP-04b; pre-clip norms were 7-30

Key signals to watch at ep5:
  - Grad/depth  > 0.1  (reproj backpropagating through depth head — KEY TEST)
  - Grad/camera > 0.05 (reproj backpropagating through camera head)
  - loss_reproj decreasing from initial ~10-50px → <10px by ep10
  - val/loss_track ≤ 0.12 (2D anchor not regressing)
  - No NaN or explosion in any grad module
